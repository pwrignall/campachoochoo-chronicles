O, joy of timestamps

A bane of a data engineer, they come back to haunt me in my free time.

From what I can gather, different cameras have different conventions for recording timestamps. None of the timestamp fields outright include timezones or timezone offsets.

The photo organiser I use, Elodie, renames files using the timestamp without offset. I would prefer to rename them using UTC time so will look to amend that at some point.

From EXIF v2.31 it appears that some convention was standardised to include the offset in a separate field, but from my limited sample size this has varying degrees of adoption. The Google Pixel follows the standard but the Fujifilm X100T does not (at least on the firmware I‚Äôm running and not in a field that is obvious to me).

Not sure if you‚Äôve set the timezone correctly on your camera? Good luck! It‚Äôs a bit of a guess for me to figure out what setting I had (e.g. if I‚Äôd changed the time to local time on travels) for photos I took a while ago. For my future self, it‚Äôs probably simplest to set the camera‚Äôs time to UTC and not worry about recording what the local time was e.g. during travelling or daylight savings.

Ensuring a good timestamp merge

The GPS timestamps from GPSLogger are in UTC and the standard for EXIF is to have the GPS timestamp in UTC also. So a basic approach to take on photos without any timezone offset information is to convert EXIF timestamps to the local time and then use that known offset when doing the -geotag merge.

This might need a bit of investigation and trial and error i.e. making a change, doing a dry-run geotag, looking up the resolved coordinates, and trying to remember if the photo looks like it was taken where it resolved to.

Show all timestamp info (source):

exiftool -time:all -G1 -a -s image.jpg

Shift all timestamps forward by an hour:

exiftool "-AllDates+=1" image.jpg

Merge with the UTC offset defined for -geotime:

exiftool -geotag "~/gps/*" "-geotime<${DateTimeOriginal}+HH:MM" *.jpg

The Reddit post I found the -geotime information on also has suggestions for no interpolation (-api GeoMaxIntSecs=0) and using the closest found coordinate within a set time window (-api GeoMaxExtSecs=120 for 2 minutes) which seem like good options to consider.

This is one of those situations where I think ‚Äògood enough‚Äô is probably enough. The results may not be 100% precise but if they end up being in the right city/country, that‚Äôll do for me.
