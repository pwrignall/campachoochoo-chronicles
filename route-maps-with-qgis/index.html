How to create an airline route map with QGIS

Wed 7 Feb 2024

This guide will take you through the steps to plot a route map in QGIS, with lines between points as great circles.

When I worked at an aviation news and data company called FlightGlobal, we occasionally wanted to display a map showing newly-announced routes, the geographical reach of a carrier, or the top n routes in a region.

For static maps, intended to be printed, I used the excellent geographic information system application, QGIS. Whilst my process most certainly could be streamlined, I‚Äôm laying it out here so if you ever feel like mapping some routes, you can too!

Data prep

Route maps tend to use great circles to connect points on a map. Because of things like weather these are not the exact coordinates an aircraft would follow, but they‚Äôre used because they‚Äôre the shortest distance between two points. And they look nice.

I had an R script I used to use for this. But I‚Äôve lost it. My attempt to re-write it in Python took a while but, thanks to the geographiclib package and its examples, I got something working that does a decent job.

The script is on GitHub with instructions and takes a couple of inputs to produce two outputs: a file containing unique airports, airport_points.csv, and a file containing all the points needed to create the great circle lines in QGIS, route_points.csv.

Adding data layers to QGIS

Once you‚Äôve installed QGIS, you‚Äôll need to add your data as layers in the application. Once all of your data layers are in, you can experiment with different styles and map projections.

Map base layer

For this I recommend the brilliant Natural Earth project. Medium scale should be fine for this use case, so you can navigate to the 1:50m Physical downloads and save the Land layer ‚Äî which includes polygons for land and major islands of the world ‚Äî to a working directory on your device.

  1. In QGIS, in an empty project, add that layer through the menu options: Layer ‚Üí Add Layer ‚Üí Add Vector Layer‚Ä¶

  2. By Vector Dataset(s), browse to your downloaded Natural Earth .zip file and then click Add. Close this dialog box.

Route points

Now we‚Äôll use the outputs from the earlier script and add them as another layer to the project. For this example, I‚Äôve plotted the hypothetical route travelled by the Champions League trophy since 1993, from the venue of that year‚Äôs final in Munich (MUC) through all the winners‚Äô nearest airports and the other final venues.

  1. Click Layer ‚Üí Add Layer ‚Üí Add Delimited Text Layer‚Ä¶

  2. By File Name, browse to your working directory and select the route_points.csv file.

  3. Under File Format, verify CSV is selected.

  4. Under Geometry Definition, select EPSG:4326 - WGS 84 as the Geometry CRS and click Add. You can then close this dialog.

  5. With the route_points layer selected, find the Zoom to Layer button on the toolbar or within the View menu to zoom in and focus on your routes, if you like.

Points to lines

Whilst there are QGIS plugins to convert point pairs to lines, I don‚Äôt think they had all the features I was looking for. I may revisit this and see if Beeline (which led me to the Python functions I used in my own script) can offer a simpler alternative to what I‚Äôm doing. Instead, I‚Äôve used a plugin called Points2One to transform the long list of longitude/latitude coordinate sequences to a set of lines.

Install the plugin through the menu, Plugins ‚Üí Manage and Install Plugins‚Ä¶ and search for ‚ÄúPoints2One‚Äù.

  1. Select the Points2One dialog through Vector ‚Üí Points2One ‚Üí Points2One.

  2. Input vector layer should be route_points. You‚Äôll want to Create lines and Group features by route. Enter an Output shapefile filename in which your lines will be saved and check the Add result to canvas box:

  1. Click OK and Close on the Points2One dialog.

Airport points

Our final layer will be marks for the individual airports. This was the other output file generated by the earlier script.

  1. Click Layer ‚Üí Add Layer ‚Üí Add Delimited Text Layer‚Ä¶

  2. By File Name, browse to your working directory and select the airport_points.csv file.

  3. Ensure File Format is CSV and Geometry CRS is EPSG:4326 - WGS 84. Click Add and close the dialog. Because our route points include the airports as start and end points, this addition won‚Äôt be obvious until we adjust the styling.

Changing the map‚Äôs appearance

Layer symbology

Editing symbology, or how layers are styled ‚Äî sometimes based on some data attribute like you would do on a choropleth ‚Äî can be done by double-clicking a layer in QGIS. You can choose an in-built style or make up your own. You can modify object fills and strokes and that‚Äôs all I‚Äôll be doing for this map, using Simple Fill, Simple Line and Simple Marker depending on the layer type.

You can also adjust the canvas colour through Project ‚Üí Properties ‚Üí General ‚Üí Background color.

Toggle a layer‚Äôs visibility completely by clicking the eye-shaped icon next to a layer. We want to do this now for the route_points layer.

Airport labels

We can add text labels to give the viewer an idea of which airport is where.

Double-click the airport_points layer then select Labels. Choose Single Labels from the dropdown. For Value I‚Äôm using the Expression Builder next to the field dropdown and have entered "iata_code" || ' ' || "loc_name" which will concatenate the airport code and municipality (maybe not the most intuitive name in all cases, as you‚Äôll see) from the airports input file.

Within this dialog you can adjust the label font, placement and any text effects like buffers and shadows.

Map projection

When the geographic extent of a map is limited to a specific region, it‚Äôs a good opportunity to use a different map projection to minimise the distortions we make when flattening a sphere onto a rectangle.

My go-to tool for choosing a good projection is Projection Wizard. With it, we can specify the boundaries of the map and receive some suggested projections in return. For this region, I‚Äôll optimise by equal area and use an Albers equal-area conic projection. The tool also gives the coordinate string you need for QGIS in either proj4 or WKT formats. QGIS accepts both but prefers WKT.

Within QGIS, define a new projection from Settings ‚Üí Custom Projections‚Ä¶ Click the plus icon to add a new projection, give it a name and paste the text from Projection Wizard into the Parameters box. Click OK.

Now re-project your map to the newly-defined projection by selecting Project ‚Üí Properties‚Ä¶. In the dialog click CRS (Coordinate Reference System). Within Predefined Coordinate Reference Systems scroll down until you see some under User-defined and select the name you just created. Click OK and there‚Äôs your re-projected map!

From here you can export the map as an image or PDF to further tweak it. Here‚Äôs my end product, after exporting and adding a title. I also took the opportunity to edit airport_points.csv and tidy up the location name labels.

Summary

And that‚Äôs it, although quite an involved process end-to-end this doesn‚Äôt take too long once you‚Äôre used to it. Except maybe choosing the colours. With this method you can create quite detailed explanatory maps as well as more abstract ones like this. Enjoy!
